# p0-w2-d3-t2 · Бейдж на иконке и липкая шапка

> Тип: новый материал · Репозиторий: один отдельный

## Задача

Два приёма, которые встречаются в каждом интерфейсе: счётчик-бейдж в углу иконки (как
непрочитанные уведомления) и шапка, которая прилипает к верху при прокрутке. Оба строятся на
позиционировании: бейдж — `absolute` внутри `relative`-родителя, шапка — `sticky`.

```
родитель: position: relative
  ┌───────────┐
  │   icon    │ (•) ← бейдж: absolute, от угла родителя
  └───────────┘

шапка: position: sticky; top: 0
едет с потоком, у верха окна — прилипает
```

## Что нужно сделать

Собрать страницу, где есть иконка с бейджем в углу и шапка, прилипающая при скролле. Контента на
странице должно быть достаточно, чтобы её можно было прокрутить и увидеть поведение шапки.

## Что нужно получить

- иконка (или кнопка) с маленьким бейджем, привязанным к её углу: бейдж — `absolute`, родитель —
  `relative`, координаты от угла родителя;
- бейдж держится в углу, даже если иконку подвигать по странице;
- шапка сайта `position: sticky; top: 0` — при прокрутке остаётся видимой сверху;
- страница достаточно длинная, чтобы прокрутка реально происходила.

## Критерии приёмки

* Бейдж позиционируется относительно своей иконки (родитель `relative`), а не относительно окна
  или всей страницы.
* Шапка прилипает к верху при прокрутке и не перекрывает контент криво.
* Можешь объяснить, почему `absolute`-бейджу нужен `relative`-родитель — что будет, если его нет.
